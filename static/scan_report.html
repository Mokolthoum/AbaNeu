Detailed Findings

CRITICAL   SQL Injection — Login Bypass
POST /login/
The login form is vulnerable to SQL Injection via raw SQL query with string interpolation. An attacker can bypass authentication using payloads like ' OR '1'='1' -- in the username/password fields. The application uses f"SELECT ... WHERE username = '{username}' AND password = '{password}'" instead of parameterized queries.
OWASP: A03:2021 — Injection
✅ Remediation: Use Django ORM User.objects.filter() or parameterized queries cursor.execute(sql, [params]) instead of string formatting in SQL queries.
CRITICAL   SQL Injection — Search Function
GET /search/?q=
The transaction search function uses raw SQL with string interpolation: WHERE description LIKE '%{query}%'. An attacker can extract data from the database using UNION-based or blind SQL injection techniques.
✅ Remediation: Use Django ORM: Transaction.objects.filter(description__icontains=query)
HIGH   Reflected XSS — Multiple Endpoints
/search/
/transfer/
/support/
/documents/view/
/profile/edit/
Multiple forms reflect user input without proper sanitization. The search page renders the query parameter using the |safe template filter, allowing injection of arbitrary JavaScript. Support tickets and profile bio fields also store and render unescaped HTML.
OWASP: A03:2021 — Injection (XSS)
✅ Remediation: Remove |safe filter from templates. Use Django's auto-escaping. Implement Content-Security-Policy header. Sanitize all user inputs with bleach or django.utils.html.escape().
HIGH   Unrestricted File Upload
POST /profile/edit/
POST /documents/upload/
Both the profile picture upload and document upload endpoints accept files of any type without validation. An attacker can upload .php, .exe, or .html files containing malicious code. The scanner successfully uploaded test_scan.php to both endpoints.
OWASP: A04:2021 — Insecure Design
✅ Remediation: Implement file extension whitelist (allow only .jpg, .png, .pdf, .txt). Validate file content type using magic bytes. Store uploads outside the web root. Rename uploaded files with random names.
MEDIUM   Missing CSRF Protection
/login/
/transfer/
/support/
/profile/edit/
/documents/upload/
5 POST forms lack CSRF tokens. The most critical is the money transfer form at /transfer/ — an attacker can craft a malicious page that submits a money transfer request on behalf of an authenticated user without their knowledge.
OWASP: A01:2021 — Broken Access Control
✅ Remediation: Enable Django's CSRF middleware in settings.py. Add {% csrf_token %} inside every <form> tag. Set CSRF_COOKIE_HTTPONLY = True.
