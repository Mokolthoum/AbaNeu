1. نظرة عامة على المشروع

VulnBank هو تطبيق ويب يُحاكي نظام بنكي حقيقي، تم بناؤه بثغرات أمنية مقصودة ليكون بمثابة "موقع هدف" (Target Website) لاختبار وعرض قدرات الماسح الأمني (Web Vulnerability Scanner) الذي تم تطويره كمشروع تخرج.

⚠️ تحذير: هذا التطبيق مُعد للاختبار الأمني المحلي فقط. يحتوي على ثغرات حقيقية ويجب عدم نشره على أي سيرفر متصل بالإنترنت.

الهدف من المشروع

  • توفير بيئة واقعية لاختبار الماسح الأمني
  • دعم جميع مراحل الفحص: الاستطلاع، الزحف، تحليل الثغرات
  • تضمين ثغرات متنوعة: حقن SQL، XSS، تضمين ملفات محلية، رفع ملفات غير مقيد
  • محاكاة بنية موقع حقيقي بمسارات مخفية وملفات حساسة مكشوفة

الوظائف الرئيسية

  • نظام تسجيل دخول وتسجيل خروج كامل
  • لوحة تحكم تعرض الرصيد والمعاملات الأخيرة
  • نموذج تحويل أموال بين الحسابات
  • بحث في المعاملات المالية
  • صفحة ملف شخصي مع رفع صورة
  • نظام دعم فني بالتذاكر
  • عارض مستندات ورفع ملفات
  • واجهات API داخلية مكشوفة

4. النماذج (Models) - قاعدة البيانات

يحتوي الملف bank/models.py على 5 نماذج تمثل جداول قاعدة البيانات:

4.1 نموذج UserProfile (الملف الشخصي)

يُوسّع نموذج المستخدم المدمج في Django بإضافة حقول: السيرة الذاتية (bio)، الهاتف، العنوان، صورة الملف الشخصي، رقم الهوية الوطنية.
ملاحظة أمنية: حقل bio هو هدف لثغرة XSS المُخزّن.

4.2 نموذج Account (الحساب البنكي)

الحقلالنوعالوصف
account_numberCharFieldرقم الحساب (فريد)
account_typeCharFieldنوع الحساب: توفير / جاري / تجاري
balanceDecimalFieldالرصيد الحالي
userForeignKeyربط بالمستخدم
is_activeBooleanFieldحالة الحساب

4.3 نموذج Transaction (المعاملات المالية)

يُسجل كل عملية تحويل مالية بين حسابين، مع المبلغ والوصف والرقم المرجعي والتاريخ.

4.4 نموذج SupportTicket (تذاكر الدعم)

يُخزّن تذاكر الدعم الفني. حقلا subject و message هما أهداف لثغرة Stored XSS لأنهما يُعرضان باستخدام فلتر |safe.

4.5 نموذج UploadedDocument (المستندات المرفوعة)

يُخزّن الملفات المرفوعة بدون أي تحقق من نوع الملف أو امتداده — ثغرة Unrestricted File Upload.

5. العروض (Views) - المنطق البرمجي

الملف bank/views.py يحتوي على جميع دوال العرض مع الثغرات المقصودة:

5.1 تسجيل الدخول - login_view()

🔴 ثغرة: SQL Injection
يستخدم استعلام SQL خام مع تنسيق النصوص (f-string) بدلاً من الاستعلامات المُعاملة: 
cursor = connection.cursor()
query = f"SELECT id, username FROM auth_user WHERE username = '{username}' AND password = '{password}'"
cursor.execute(query)

5.2 لوحة التحكم - dashboard()

تعرض الرصيد الإجمالي، الحسابات المرتبطة، وآخر 10 معاملات مالية. تتطلب تسجيل الدخول.

5.3 تحويل الأموال - transfer_money()

🔵 هدف لـ Form Analysis: نموذج تحويل أموال بدون حماية CSRF (تم تعطيل المكون الوسيط CsrfViewMiddleware في الإعدادات).

5.4 البحث في المعاملات - search_transactions()

🔴 ثغرتان:
  1. SQL Injection: استعلام SQL خام مع إدخال المستخدم في جملة LIKE
  2. Reflected XSS: استعلام البحث يُعرض في الصفحة باستخدام فلتر |safe
sql = f"""SELECT ... FROM bank_transaction t
WHERE t.description LIKE '%{query}%'
   OR t.reference_number LIKE '%{query}%' ..."""

وفي القالب:

Showing results for: <strong>{{ query|safe }}</strong>

5.5 الملف الشخصي - user_profile()

🔴 ثغرة: Stored XSS
حقل السيرة الذاتية (bio) يُعرض باستخدام {{ profile.bio|safe }} مما يسمح بتنفيذ أكواد JavaScript مُخزّنة.

5.6 الدعم الفني - support_page()

🔴 ثغرة: Stored XSS
عنوان ورسالة التذكرة يُعرضان باستخدام |safe.

5.7 عارض المستندات - view_document()

🔴 ثغرة: Local File Inclusion (LFI)
يقرأ الملف من القرص مباشرة بناءً على معامل file بدون أي تحقق: 
file_path = os.path.join(settings.BASE_DIR, 'documents', file_param)
with open(file_path, 'r') as f:
    content = f.read()

5.8 رفع الملفات - upload_document()

🔴 ثغرة: Unrestricted File Upload
لا يوجد أي تحقق من امتداد الملف أو نوع المحتوى. يمكن رفع أي ملف بما في ذلك ملفات تنفيذية.

8. الثغرات المقصودة (Intentional Vulnerabilities)

فيما يلي جدول شامل بجميع الثغرات المُدمجة، مع مواقعها وطريقة تنفيذها:

الثغرةالموقعالآليةاختبار سريع
SQL Injection صفحة تسجيل الدخول
/login/		 استعلام SQL خام مع f-string في connection.cursor().execute() أدخل ' OR 1=1 -- في حقل اسم المستخدم
SQL Injection بحث المعاملات
/search/?q=		 إدخال المستخدم مباشرة في جملة LIKE بدون تطهير أدخل ' UNION SELECT 1,2,3,4,5,6,7-- في حقل البحث
Reflected XSS نتائج البحث
/search/?q=		 الاستعلام يُعرض بـ {{ query|safe }} ابحث عن:  <script>alert('XSS')</script>
Stored XSS الملف الشخصي
/profile/		 Bio يُعرض بـ {{ profile.bio|safe }} عدّل Bio وأضف: <script>alert('XSS')</script>
Stored XSS تذاكر الدعم
/support/		 Subject و Message يُعرضان بـ |safe أنشئ تذكرة بعنوان يحتوي على HTML/JS
LFI عارض المستندات
/documents/view/		 open() بدون تحقق من path traversal جرّب: ?file=../../../etc/passwd
Unrestricted Upload رفع الملفات
/documents/upload/		 لا يوجد تحقق من الامتداد أو نوع المحتوى ارفع ملف .php أو .exe
Insecure Headers settings.py حذف SecurityMiddleware و CsrfViewMiddleware و XFrameOptions افحص الترويسات في DevTools → Network
Info Disclosure APIs مكشوفة /api/v1/internal/* بدون مصادقة افتح /api/v1/internal/users/ في المتصفح

9. ملفات الاستطلاع (Reconnaissance Files)

9.1 ملف .env.example

يحتوي على بيانات حساسة مزيفة:

  • المفتاح السري لـ Django
  • كلمات مرور قاعدة البيانات
  • مفاتيح AWS (Access Key + Secret Key)
  • مفاتيح Stripe للدفع
  • مفاتيح API داخلية

9.2 ملف database_schema.sql

يكشف هيكل قاعدة البيانات الكامل بما في ذلك أسماء الجداول والأعمدة.

9.3 ملف robots.txt

يحتوي على قائمة بالمسارات المخفية مما يُسهّل على الماسح اكتشافها.

9.4 مجلد backup/

يحتوي على db_backup.json مع بيانات المستخدمين والحسابات.

9.5 مجلد dev_logs/

يحتوي على debug.log مع stack traces وأخطاء تكشف مسارات داخلية.

12. طريقة التشغيل

المتطلبات

  • Python 3.10 أو أحدث

خطوات التشغيل

# 1. إنشاء بيئة افتراضية
python3 -m venv venv
source venv/bin/activate    # على Mac/Linux
venv\Scripts\activate       # على Windows

# 2. تثبيت المكتبات
pip install -r requirements.txt

# 3. إنشاء قاعدة البيانات
python manage.py migrate

# 4. إضافة البيانات التجريبية

# 5. تشغيل السيرفر
python manage.py runserver 8000
